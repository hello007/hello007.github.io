## 1. LangChain4j

### 🎯 掌握程度：必须掌握

### 什么是LangChain4j？

**通俗解释：**
LangChain4j是"AI应用的Spring Boot"。就像Spring Boot简化了Web开发，LangChain4j简化了AI应用开发。

**技术定义：**
Java语言的AI应用开发框架，提供LLM调用、RAG、Agent、Prompt管理等完整功能。

### 为什么选择LangChain4j？

```
LangChain4j的优势：
✅ 原生Java：纯Java实现，不是Python移植
✅ 功能完善：覆盖RAG、Agent、Memory等
✅ 易于集成：支持Spring Boot、Quarkus
✅ 性能优秀：针对JVM优化
✅ 社区活跃：快速迭代，文档完善
```

### RAG实现

```java
@Service
public class RAGService {

    private ChatLanguageModel chatModel;
    private EmbeddingModel embeddingModel;
    private EmbeddingStore<TextSegment> vectorStore;

    // 构建知识库
    public void buildKnowledgeBase(List<Document> documents) {

        // 1. 创建EmbeddingModel
        embeddingModel = OpenAiEmbeddingModel.builder()
            .apiKey(System.getenv("OPENAI_API_KEY"))
            .build();

        // 2. 创建VectorStore（内存存储）
        vectorStore = new InMemoryEmbeddingStore<>();

        // 3. 文档分块和向量化
        for (Document doc : documents) {
            List<TextSegment> chunks = splitDocument(doc, 500);

            for (TextSegment chunk : chunks) {
                // 向量化
                Embedding embedding = embeddingModel.embed(chunk).content();

                // 存储
                vectorStore.add(embedding, chunk);
            }
        }
    }

    // RAG问答
    public String answerQuestion(String question) {

        // 1. 问题向量化
        Embedding queryEmbedding = embeddingModel.embed(question).content();

        // 2. 检索相关文档
        List<EmbeddingMatch<TextSegment>> relevant =
            vectorStore.findRelevant(queryEmbedding, 5);

        // 3. 构建Prompt
        String context = relevant.stream()
            .map(match -> match.embedded().text())
            .collect(Collectors.joining("\n\n"));

        String prompt = String.format("""
            基于以下文档回答问题：

            %s

            问题：%s
            """, context, question);

        // 4. 生成答案
        return chatModel.generate(prompt);
    }
}
```

```java
@Service
public class RAGService {

    @Autowired
    private ChatClient chatClient;

    @Autowired
    private EmbeddingClient embeddingClient;

    @Autowired
    private VectorStore vectorStore;

    // 文档加载和向量化
    public void loadDocuments(List<Document> documents) {

        for (Document doc : documents) {
            // 向量化
            List<Float> embedding = embeddingClient.embed(
                doc.getContent()
            );

            // 存储
            doc.setEmbedding(embedding);
            vectorStore.add(doc);
        }
    }

    // RAG问答
    public String query(String question) {

        // 1. 检索
        List<Document> relevantDocs = vectorStore.similaritySearch(
            SearchRequest.query(question).withTopK(5)
        );

        // 2. 构建Prompt
        String context = relevantDocs.stream()
            .map(Document::getContent)
            .collect(Collectors.joining("\n\n"));

        // 3. 生成答案
        return chatClient.call(
            Prompt.from(String.format(
                "基于以下文档回答问题：\n\n%s\n\n问题：%s",
                context,
                question
            ))
        );
    }
}
```

## 3. API Rate Limiting（速率限制）

### 🎯 掌握程度：必须掌握

### 什么是API Rate Limiting？

**通俗解释：**
Rate Limiting就是"调用频率控制"。就像限速标志限制车速，API Rate Limiting限制你调用LLM API的频率，防止超限被罚。

**技术定义：**
控制API调用速率的技术手段，用于防止超出配额、控制成本、保证稳定性。

### 为什么需要Rate Limiting？

```
问题：
1. 超出配额
   - OpenAI: 每分钟Token限制
   - 超限后API调用失败

2. 成本失控
   - 无限调用导致费用爆炸
   - 需要预算控制

3. 服务不稳定
   - 突发流量导致超时
   - 需要平滑请求
```

### Rate Limiting策略

#### 策略1：固定速率（Token Bucket）

```java
@Service
public class RateLimiterService {

    // Guava RateLimiter
    private final RateLimiter rateLimiter =
        RateLimiter.create(10.0); // 每秒10个请求

    public String callLLMWithRateLimit(String prompt) {

        // 等待获取许可
        rateLimiter.acquire();

        // 调用API
        return llmService.chat(prompt);
    }
}
```

#### 策略2：滑动窗口

```java
@Service
public class SlidingWindowRateLimiter {

    private final int maxRequests = 100;  // 最大请求数
    private final long windowSize = 60000; // 时间窗口：60秒

    private final ConcurrentLinkedDeque<Long> timestamps =
        new ConcurrentLinkedDeque<>();

    public synchronized boolean tryAcquire() {

        long now = System.currentTimeMillis();

        // 移除窗口外的记录
        while (!timestamps.isEmpty() &&
               now - timestamps.peekFirst() > windowSize) {
            timestamps.removeFirst();
        }

        // 检查是否超限
        if (timestamps.size() >= maxRequests) {
            return false;
        }

        timestamps.addLast(now);
        return true;
    }

    public String callLLM(String prompt) {

        if (!tryAcquire()) {
            throw new RateLimitException("调用频率超限，请稍后重试");
        }

        return llmService.chat(prompt);
    }
}
```

#### 策略3：分布式限流（Redis）

```java
@Service
public class DistributedRateLimiter {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String KEY_PREFIX = "rate_limit:";

    public boolean tryAcquire(String apiKey, int maxRequests, int windowSeconds) {

        String key = KEY_PREFIX + apiKey;

        // Lua脚本：原子操作
        String luaScript = """
            local current = redis.call('GET', KEYS[1])
            if current == false then
                redis.call('SET', KEYS[1], 1)
                redis.call('EXPIRE', KEYS[1], ARGV[1])
                return 1
            elseif tonumber(current) < tonumber(ARGV[2]) then
                redis.call('INCR', KEYS[1])
                return 1
            else
                return 0
            end
            """;

        Long result = redisTemplate.execute(
            RedisScript.of(luaScript, Long.class),
            Collections.singletonList(key),
            String.valueOf(windowSeconds),
            String.valueOf(maxRequests)
        );

        return result == 1;
    }
}
```

### Token级别限流

```java
@Service
public class TokenRateLimiter {

    private final AtomicInteger currentTokens = new AtomicInteger(0);
    private final int maxTokensPerMinute = 150000;

    @Scheduled(fixedRate = 60000) // 每分钟重置
    public void resetTokens() {
        currentTokens.set(0);
    }

    public String callLLM(String prompt) {

        // 估算Token数
        int estimatedTokens = estimateTokens(prompt);

        // 检查是否超限
        if (currentTokens.get() + estimatedTokens > maxTokensPerMinute) {
            throw new RateLimitException(
                "Token使用超限，请等待下一分钟"
            );
        }

        // 更新计数
        currentTokens.addAndGet(estimatedTokens);

        // 调用API
        return llmService.chat(prompt);
    }
}
```

### 实际应用

```java
@RestController
@RequestMapping("/api/ai")
public class AIController {

    @Autowired
    private RateLimiterService rateLimiter;

    @Autowired
    private LLMService llmService;

    @PostMapping("/chat")
    public ResponseEntity<?> chat(@RequestBody ChatRequest request) {

        try {
            // 尝试获取许可
            if (!rateLimiter.tryAcquire(
                request.getApiKey(),
                100,  // 每分钟100次
                60    // 60秒窗口
            )) {
                return ResponseEntity.status(429)
                    .body("请求过于频繁，请稍后重试");
            }

            // 调用LLM
            String response = llmService.chat(request.getMessage());
            return ResponseEntity.ok(response);

        } catch (RateLimitException e) {
            return ResponseEntity.status(429)
                .body(e.getMessage());
        }
    }
}
```

## 4. Caching Strategy（缓存策略）

### 🎯 掌握程度：必须掌握

### 为什么需要缓存？

```
问题：
├─ 重复问题重复调用
│   "什么是Java？" → 每次都调用LLM
│   成本高、速度慢
│
└─ 相似问题可以复用
    "Java是什么？"和"什么是Java？"
    答案一样，不需要重复调用

解决方案：
├─ 精确缓存：问题完全一样，直接返回缓存
├─ 语义缓存：问题意思一样，返回缓存
└─ 降低成本、提升速度
```

### 缓存策略

#### 策略1：精确缓存（Redis）

```java
@Service
public class ExactCacheService {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    private static final String CACHE_PREFIX = "llm_cache:";

    public String chat(String message) {

        // 计算缓存key
        String cacheKey = CACHE_PREFIX +
            DigestUtils.md5DigestAsHex(message.getBytes());

        // 尝试从缓存获取
        String cached = redisTemplate.opsForValue().get(cacheKey);
        if (cached != null) {
            log.info("缓存命中");
            return cached;
        }

        // 调用LLM
        String response = llmService.chat(message);

        // 存入缓存（24小时）
        redisTemplate.opsForValue().set(
            cacheKey,
            response,
            24,
            TimeUnit.HOURS
        );

        return response;
    }
}
```

#### 策略2：语义缓存（向量相似度）

```java
@Service
public class SemanticCacheService {

    @Autowired
    private EmbeddingModel embeddingModel;

    @Autowired
    private VectorStore cacheStore;

    public String chat(String question) {

        // 1. 问题向量化
        Embedding queryEmbedding =
            embeddingModel.embed(question).content();

        // 2. 检索相似问题
        List<EmbeddingMatch<CachedResponse>> matches =
            cacheStore.findRelevant(queryEmbedding, 1);

        // 3. 如果相似度>0.95，返回缓存
        if (!matches.isEmpty() &&
            matches.get(0).score() > 0.95) {

            CachedResponse cached = matches.get(0).embedding();
            log.info("语义缓存命中，相似度: {}", matches.get(0).score());
            return cached.getResponse();
        }

        // 4. 调用LLM
        String response = llmService.chat(question);

        // 5. 存入语义缓存
        cacheStore.add(
            queryEmbedding,
            new CachedResponse(question, response)
        );

        return response;
    }
}
```

#### 策略3：多级缓存

```java
@Service
public class MultiLevelCacheService {

    // L1：本地缓存（Caffeine）
    private Cache<String, String> localCache = Caffeine.newBuilder()
        .maximumSize(1000)
        .expireAfterWrite(10, TimeUnit.MINUTES)
        .build();

    @Autowired
    private RedisTemplate<String, String> redisTemplate; // L2：Redis

    @Autowired
    private VectorStore semanticCache; // L3：语义缓存

    public String chat(String message) {

        // L1：本地缓存
        String cached = localCache.getIfPresent(message);
        if (cached != null) {
            return cached;
        }

        // L2：Redis精确缓存
        String redisKey = "llm:" + DigestUtils.md5DigestAsHex(message.getBytes());
        cached = redisTemplate.opsForValue().get(redisKey);
        if (cached != null) {
            localCache.put(message, cached);
            return cached;
        }

        // L3：语义缓存
        Embedding embedding = embeddingModel.embed(message).content();
        List<EmbeddingMatch<String>> matches =
            semanticCache.findRelevant(embedding, 1);

        if (!matches.isEmpty() && matches.get(0).score() > 0.95) {
            cached = matches.get(0).embedded();
            redisTemplate.opsForValue().set(redisKey, cached, 24, TimeUnit.HOURS);
            localCache.put(message, cached);
            return cached;
        }

        // 缓存未命中，调用LLM
        String response = llmService.chat(message);

        // 写入各级缓存
        localCache.put(message, response);
        redisTemplate.opsForValue().set(redisKey, response, 24, TimeUnit.HOURS);
        semanticCache.add(embedding, response);

        return response;
    }
}
```

## 5. Streaming Response（流式响应）

### 🎯 掌握程度：必须掌握

### 什么是流式响应？

**通俗解释：**
流式响应就像"打字机效果"。AI不是一次性输出全部内容，而是一个字一个字地输出，用户能实时看到AI在"思考和回答"。

**技术定义：**
服务器将响应分成多个小块，逐个发送给客户端，客户端实时接收和显示。

### 为什么需要流式响应？

```
传统响应（非流式）：
用户：请解释什么是微服务
[等待3秒...]
AI：[一次性返回完整答案]

问题：
├─ 用户体验差（长时间等待）
├─ 无法提前终止
└─ 感觉"卡顿"

流式响应：
用户：请解释什么是微服务
AI：[逐字输出] "微服务是..." → "一种架构..." → "..."
     用户实时看到AI在回答
     可以随时停止

优势：
├─ 体验好（即时反馈）
├─ 可控（随时停止）
└─ 符合人机交互习惯
```

### 前端打字机效果

```javascript
class StreamingChat {
    constructor() {
        this.fullText = '';
        this.displayedText = '';
    }

    async streamResponse(message) {
        const response = await fetch('/api/chat/stream?message=' + encodeURIComponent(message));
        const reader = response.body.getReader();
        const decoder = new TextDecoder();

        this.fullText = '';
        this.displayedText = '';

        while (true) {
            const { done, value } = await reader.read();

            if (done) break;

            const chunk = decoder.decode(value);
            this.fullText += chunk;

            // 打字机效果：逐字显示
            this.typeWriterEffect();
        }
    }

    typeWriterEffect() {
        const interval = setInterval(() => {
            if (this.displayedText.length < this.fullText.length) {
                this.displayedText += this.fullText[this.displayedText.length];
                this.updateDisplay();
            } else {
                clearInterval(interval);
            }
        }, 30); // 每30ms显示一个字符
    }

    updateDisplay() {
        document.getElementById('chat-output').textContent =
            this.displayedText;
    }
}
```

```java
@Service
public class StreamingOptimizationService {

    // 1. 批量发送：积累多个token后一次性发送
    private final StringBuilder buffer = new StringBuilder();
    private static final int BUFFER_SIZE = 10; // 缓冲10个token

    public void optimizedStream(String token, Consumer<String> sender) {

        buffer.append(token);

        if (buffer.length() >= BUFFER_SIZE) {
            sender.accept(buffer.toString());
            buffer.setLength(0);
        }
    }

    // 2. 智能分句：在句子边界发送
    public void smartStream(String token, Consumer<String> sender) {

        buffer.append(token);

        // 检测句子结束
        if (token.contains("。") || token.contains("！") ||
            token.contains("？") || token.contains("\n")) {

            sender.accept(buffer.toString());
            buffer.setLength(0);
        }
    }

    // 3. Markdown实时渲染
    public void markdownStream(String fullText, Consumer<String> sender) {

        // 实时解析Markdown
        String rendered = MarkdownRenderer.render(fullText);

        // 只发送变化部分
        String delta = getDelta(rendered);
        if (!delta.isEmpty()) {
            sender.accept(delta);
        }
    }
}
```

## 6. Orchestration（编排）

### 🎯 掌握程度：必须掌握

### 什么是Orchestration？

**通俗解释：**
编排就像"指挥家"。多个AI模型、多个工具需要协同工作，编排层负责协调它们，让整体系统有序运行。

**技术定义：**
协调多个AI组件（模型、工具、服务）协同工作的架构层。

### 编排模式

#### 模式1：链式编排（Chain）

```java
@Service
public class ParallelOrchestrator {

    // 并行调用多个模型，汇总结果
    public String analyze(String text) {

        // 并行执行多个任务
        CompletableFuture<String> sentimentFuture =
            CompletableFuture.supplyAsync(() ->
                llmService.analyzeSentiment(text)
            );

        CompletableFuture<String> keywordsFuture =
            CompletableFuture.supplyAsync(() ->
                llmService.extractKeywords(text)
            );

        CompletableFuture<String> summaryFuture =
            CompletableFuture.supplyAsync(() ->
                llmService.summarize(text)
            );

        // 等待所有任务完成
        CompletableFuture.allOf(
            sentimentFuture,
            keywordsFuture,
            summaryFuture
        ).join();

        // 汇总结果
        return String.format("""
            情感分析：%s
            关键词：%s
            摘要：%s
            """,
            sentimentFuture.join(),
            keywordsFuture.join(),
            summaryFuture.join()
        );
    }
}
```

```java
@Service

    public String process(String input) {

    }

    private String handleGeneralQuestion(String question) {
        // 一般问题：直接回答
        return llmService.chat(question);
    }

    private String handleComplaint(String complaint) {
        // 投诉：记录 → 特殊回复 → 通知
        ticketService.create(complaint);
        String response = llmService.generateEmpathyResponse(complaint);
        notificationService.alert(complaint);
        return response;
    }
```

```java
public class LoopOrchestrator {

    // 迭代优化：直到满足条件
    public String optimize(String prompt) {

        String currentVersion = llmService.generate(prompt);
        int iterations = 0;
        final int MAX_ITERATIONS = 5;

        while (iterations < MAX_ITERATIONS) {

            // 评估当前版本
            String feedback = llmService.evaluate(currentVersion);

            // 根据反馈优化
            currentVersion = llmService.improve(
                currentVersion,
                feedback
            );

            iterations++;
        }

        return currentVersion;
    }
}
```

```java
@Service
public class LangChain4jOrchestrator {

    public String orchestrate(String userQuery) {

        // 创建Chain
        Chain chain = SequentialChain.builder()

            // 第一步：分类
            .addFirst(
                "classify",
                ClassificationAction.builder()
                    .query(userQuery)
                    .build()
            )

                RoutingAction.builder()

                    .routes(Map.of(
                        "technical", technicalChain(),
                        "general", generalChain()
                    ))
                    .build()
            )

            // 第三步：格式化输出
            .addLast(
                "format",
                FormattingAction.builder()
                    .inputVariable("route.output")
                    .build()
            )
            .build();

        // 执行Chain
        Map<String, Object> result = chain.execute();

        return (String) result.get("format.output");
    }

    private Chain technicalChain() {
        // 技术问题处理链
        return SequentialChain.builder()
            .add("search", searchAction())
            .add("answer", answerAction())
            .build();
    }

    private Chain generalChain() {
        // 一般问题处理链
        return SequentialChain.builder()
            .add("chat", chatAction())
            .build();
    }
```

```
业务指标：
├─ Token使用量

质量指标：
├─ 用户满意度

└─ 幻觉率
```

### 日志记录

```java
@Service
@Slf4j
public class ObservabilityService {

    @Autowired
    private ChatLanguageModel model;

    @NewLine("\n") // 自定义注解
    public String chat(String userMessage) {

        // 记录请求
        MDC.put("request_id", UUID.randomUUID().toString());
        MDC.put("user_message", userMessage);

        long startTime = System.currentTimeMillis();

        try {

            // 调用LLM
            String response = model.generate(userMessage);

            // 记录响应
            long duration = System.currentTimeMillis() - startTime;
            int estimatedTokens = estimateTokens(userMessage + response);

            log.info("LLM call completed | duration={}ms | tokens={}",
                duration, estimatedTokens);

            // 记录到监控系统
            meterRegistry.timer("llm.duration")
                .record(duration, TimeUnit.MILLISECONDS);

            meterRegistry.counter("llm.tokens.used",
                "model", "gpt-4")
                .increment(estimatedTokens);

            return response;

        } catch (Exception e) {

            log.error("LLM call failed", e);
            meterRegistry.counter("llm.errors",
                "type", e.getClass().getSimpleName())
                .increment();
            throw e;

        } finally {
            MDC.clear();
        }
    }
}
```

```java
@Service
public class CostMonitoringService {

    private final AtomicInteger totalTokens = new AtomicInteger(0);
    private final AtomicDouble totalCost = new AtomicDouble(0.0);

    @EventListener
    public void onLLMCall(LLMCallEvent event) {

        int inputTokens = event.getInputTokens();
        int outputTokens = event.getOutputTokens();
        int total = inputTokens + outputTokens;

        // 计算成本
        double cost = calculateCost(
            event.getModel(),
            inputTokens,
            outputTokens
        );

        // 更新统计
        totalTokens.addAndGet(total);
        totalCost.addAndGet(cost);

        // 记录日志
        log.info("LLM Call | model={} | tokens={} | cost=${}",
            event.getModel(),
            total,
            String.format("%.4f", cost)
        );

        // 检查预算
        if (totalCost.get() > getBudgetLimit()) {
            log.warn("Budget limit exceeded! Total cost: ${}",
                String.format("%.2f", totalCost.get()));
            // 发送告警
            alertService.sendAlert("LLM budget exceeded");
        }
    }

    @Scheduled(cron = "0 0 * * * *") // 每小时报告
    public void reportCost() {

        int tokens = totalTokens.get();
        double cost = totalCost.get();

        log.info("===== LLM Cost Report =====");
        log.info("Total Tokens: {}", tokens);
        log.info("Total Cost: ${}", String.format("%.2f", cost));

    }
```
