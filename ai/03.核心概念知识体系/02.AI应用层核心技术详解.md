## 1. RAG（检索增强生成）

### 🎯 掌握程度：必须掌握

### 什么是RAG？

**通俗解释：**
RAG就像"开卷考试"。LLM是考生，知识库是教科书。考生回答问题时，可以翻阅教科书找到相关内容，然后基于这些内容回答问题。

**技术定义：**
Retrieval-Augmented Generation，通过检索外部知识库，增强LLM生成答案的准确性和可靠性。

### 为什么需要RAG？

#### 问题1：LLM的知识有局限

```
LLM的问题：
❌ 训练数据有截止日期
   问："2025年最新的Java版本是什么？"
   答：不知道（训练数据只到2023年）

❌ 不懂私有知识
   问："公司的报销流程是什么？"
   答：不知道

❌ 容易"幻觉"
   问："我们公司有客户支持热线吗？"
   答：可能编造一个错误号码
```

#### RAG的解决方案

```
RAG的优势：
✅ 实时知识
   检索最新文档，获取最新信息

✅ 私有知识
   检索公司内部文档，获取私有知识

✅ 可靠答案
   基于真实文档，减少幻觉

✅ 可追溯
   标注答案来源，方便验证
```

### RAG工作流程

```
┌─────────────────────────────────────────────────┐
│ RAG完整流程                                      │
└─────────────────────────────────────────────────┘

1. 文档准备阶段（离线）
   ┌──────────────┐
   │ 原始文档      │  (PDF, Word, Wiki, HTML)
   └──────┬───────┘
          ↓
   ┌──────────────┐
   │ 文档解析      │  (Tika, BeautifulSoup)
   └──────┬───────┘
          ↓
   ┌──────────────┐
   │ 文本分块      │  (Chunking: 500-1000字符)
   └──────┬───────┘
          ↓
   ┌──────────────┐
   │ 向量化       │  (Embedding API)
   └──────┬───────┘
          ↓
   ┌──────────────┐
   │ 存入向量库    │  (PgVector, Milvus)
   └──────────────┘

2. 问答阶段（在线）
   ┌──────────────┐
   │ 用户问题      │  "如何申请年假？"
   └──────┬───────┘
          ↓
   ┌──────────────┐
   │ 问题向量化    │  (Embedding)
   └──────┬───────┘
          ↓
   ┌──────────────┐
   │ 相似度检索    │  (找到相关文档片段)
   └──────┬───────┘
          ↓
   ┌──────────────┐
   │ 构建Prompt   │  (问题 + 检索内容)
   └──────┬───────┘
          ↓
   ┌──────────────┐
   │ LLM生成答案  │  (基于真实文档回答)
   └──────┬───────┘
          ↓
   ┌──────────────┐
   │ 返回答案+引用 │
   └──────────────┘
```

### RAG vs Fine-tuning

| 维度 | RAG | Fine-tuning |
|------|-----|-------------|
| **原理** | 外挂知识库 | 训练模型 |
| **成本** | 低 | 高 |
| **实现难度** | 中等 | 困难 |
| **知识更新** | 实时更新 | 需要重新训练 |
| **适用场景** | 知识密集型任务 | 特定领域适配 |
| **推荐程度** | ⭐⭐⭐⭐⭐ | ⭐⭐ |

**结论：** 99%的应用场景优先选择RAG

### Java实现RAG系统

#### 架构设计

```java
// RAG系统核心组件

@Service
public class RAGService {

    private final EmbeddingService embeddingService;
    private final VectorDatabase vectorDB;
    private final LLMAIService llmService;

    // 知识库构建（离线）
    public void buildKnowledgeBase(List<Document> documents) {
        for (Document doc : documents) {
            // 1. 解析文档
            String text = documentParser.parse(doc);

            // 2. 分块
            List<String> chunks = chunker.split(text, 500);

            // 3. 向量化
            for (String chunk : chunks) {
                float[] vector = embeddingService.embed(chunk);

                // 4. 存储到向量库
                vectorDB.insert(new VectorChunk(
                    doc.getId(),
                    chunk,
                    vector,
                    getMetadata(doc, chunk)
                ));
            }
        }
    }

    // RAG问答（在线）
    public Answer answerQuestion(String question) {
        // 1. 问题向量化
        float[] queryVector = embeddingService.embed(question);

        // 2. 检索相关文档片段
        List<VectorChunk> relevantChunks = vectorDB.search(
            queryVector,
            topK = 5  // 取最相关的5个片段
        );

        // 3. 构建Prompt
        String prompt = buildRAGPrompt(question, relevantChunks);

        // 4. LLM生成答案
        String answer = llmService.chat(prompt);

        // 5. 返回答案+来源
        return new Answer(answer, relevantChunks);
    }

    // 构建RAG Prompt
    private String buildRAGPrompt(String question,
                                   List<VectorChunk> chunks) {
        StringBuilder context = new StringBuilder();
        context.append("基于以下文档回答问题：\n\n");

        for (int i = 0; i < chunks.size(); i++) {
            VectorChunk chunk = chunks.get(i);
            context.append(String.format(
                "[文档%d] %s\n来源：%s\n\n",
                i + 1,
                chunk.getText(),
                chunk.getMetadata().getSource()
            ));
        }

        context.append(String.format(
            "问题：%s\n\n" +
            "要求：基于上述文档内容回答问题，" +
            "如果文档中没有答案，请说'文档中没有找到相关信息'。",
            question
        ));

        return context.toString();
    }
}
```

### 关键技术点

#### 1. 文档分块（Chunking）

```java
@Service
public class DocumentChunker {

    // 智能分段：保持语义完整
    public List<String> chunk(String text, int targetSize) {
        List<String> chunks = new ArrayList<>();

        // 按段落分割
        String[] paragraphs = text.split("\n\n");

        StringBuilder currentChunk = new StringBuilder();

        for (String paragraph : paragraphs) {
            if (currentChunk.length() + paragraph.length() > targetSize) {
                if (currentChunk.length() > 0) {
                    chunks.add(currentChunk.toString());
                }
                currentChunk = new StringBuilder(paragraph);
            } else {
                currentChunk.append(paragraph).append("\n\n");
            }
        }

        if (currentChunk.length() > 0) {
            chunks.add(currentChunk.toString());
        }

        return chunks;
    }

    // 重叠分块：保留上下文
    public List<String> chunkWithOverlap(String text,
                                          int chunkSize,
                                          int overlap) {
        List<String> chunks = new ArrayList<>();
        int start = 0;

        while (start < text.length()) {
            int end = Math.min(start + chunkSize, text.length());
            chunks.add(text.substring(start, end));
            start = end - overlap;
        }

        return chunks;
    }
}
```

### RAG优化策略

#### 策略1：混合检索

```
向量检索 + 关键词检索

向量检索：
├─ 语义理解
├─ 找到相关内容
└─ 可能遗漏关键词

关键词检索：
├─ 精确匹配
├─ 确保不遗漏
└─ 不理解语义

混合：取并集 → 重排序 → 最佳结果
```

#### 策略2：重排序（Reranking）

```java
// 初步检索：召回100个候选
List<VectorChunk> candidates = vectorDB.search(queryVector, 100);

// 重排序：使用更精确的模型
List<VectorChunk> reranked = reranker.rerank(
    query,
    candidates,
    topK = 10  // 只保留前10个最相关的
);
```

#### 策略3：查询扩展

## 2. Vector Database（向量数据库）

### 🎯 掌握程度：必须掌握

### 什么是向量数据库？

**通俗解释：**
向量数据库是"专门存文本语义的数据库"。传统数据库存关键词，向量数据库存"意思"。

**技术定义：**
专门用于存储、索引和查询高维向量的数据库，支持高效的相似度搜索。

### 为什么需要向量数据库？

```
传统数据库的问题：
┌────────────────────────────────┐
│ MySQL/PostgreSQL               │
├────────────────────────────────┤
│ SELECT * FROM docs             │
│ WHERE content LIKE '%苹果%'     │ ← 只能精确匹配
└────────────────────────────────┘

查询："手机" → 找不到 "iPhone" ❌

向量数据库的优势：
┌────────────────────────────────┐
│ Vector Database                │
├────────────────────────────────┤
│ SEARCH similar TO "手机"       │ ← 语义理解
└────────────────────────────────┘

查询："手机" → 找到 "iPhone" ✅
        → 找到 "华为" ✅
        → 找到 "智能手机" ✅
```

### 向量数据库对比

| 数据库 | 推荐度 | 优势 | 劣势 | 适用场景 |
|--------|-------|------|------|----------|
| **PgVector** | ⭐⭐⭐⭐⭐ | PostgreSQL插件，零额外成本 | 性能略逊于专业向量库 | 已有PostgreSQL，中小规模 |
| **Milvus** | ⭐⭐⭐⭐ | 专业向量库，性能好 | 需要独立部署，运维复杂 | 大规模生产环境 |
| **Redis Stack** | ⭐⭐⭐⭐ | 兼顾缓存和向量 | 需要Redis | 已有Redis，需要缓存+向量 |
| **Chroma** | ⭐⭐⭐ | 易用，Python生态 | Java集成较复杂 | 快速原型 |
| **SimpleVectorDB** | ⭐⭐⭐ | 内存向量，零依赖 | 不适合大规模 | 学习测试，小规模应用 |

#### Java集成

```java
@Repository
public class VectorRepository {

    @Autowired
    private JdbcTemplate jdbcTemplate;

    // 插入向量
    public void insert(String documentId, String text,
                      float[] embedding, String metadata) {
        String sql = """
            INSERT INTO document_chunks
            (document_id, chunk_text, embedding, metadata)
            VALUES (?, ?, ?, ?::jsonb)
            """;

        jdbcTemplate.update(sql,
            documentId,
            text,
            new PGvector(embedding),  // 需要自定义类型
            metadata
        );
    }

    // 相似度搜索
    public List<DocumentChunk> search(float[] queryEmbedding,
                                      int topK) {
        String sql = """
            SELECT id, document_id, chunk_text, metadata,
                   1 - (embedding <=> ?) as similarity
            FROM document_chunks
            ORDER BY embedding <=> ?
            LIMIT ?
            """;

        return jdbcTemplate.query(sql,
            new Object[]{
                new PGvector(queryEmbedding),
                new PGvector(queryEmbedding),
                topK
            },
            (rs, rowNum) -> new DocumentChunk(
                rs.getLong("id"),
                rs.getString("document_id"),
                rs.getString("chunk_text"),
                rs.getDouble("similarity"),
                rs.getString("metadata")
            )
        );
    }

    // 带过滤的搜索
    public List<DocumentChunk> searchWithFilter(
        float[] queryEmbedding,
        int topK,
        String category
    ) {
        String sql = """
            SELECT id, document_id, chunk_text, metadata,
                   1 - (embedding <=> ?) as similarity
            FROM document_chunks
            WHERE metadata->>'category' = ?
            ORDER BY embedding <=> ?
            LIMIT ?
            """;

        return jdbcTemplate.query(sql,
            new Object[]{
                new PGvector(queryEmbedding),
                category,
                new PGvector(queryEmbedding),
                topK
            },
            (rs, rowNum) -> new DocumentChunk(
                rs.getLong("id"),
                rs.getString("document_id"),
                rs.getString("chunk_text"),
                rs.getDouble("similarity"),
                rs.getString("metadata")
            )
        );
    }
}
```

### 向量相似度计算

```java
public class SimilarityCalculator {

    // 余弦相似度（最常用）
    public static double cosineSimilarity(float[] v1, float[] v2) {
        double dotProduct = 0.0;
        double norm1 = 0.0;
        double norm2 = 0.0;

        for (int i = 0; i < v1.length; i++) {
            dotProduct += v1[i] * v2[i];
            norm1 += v1[i] * v1[i];
            norm2 += v2[i] * v2[i];
        }

        return dotProduct / (Math.sqrt(norm1) * Math.sqrt(norm2));
    }

    // 欧氏距离
    public static double euclideanDistance(float[] v1, float[] v2) {
        double sum = 0.0;
        for (int i = 0; i < v1.length; i++) {
            sum += Math.pow(v1[i] - v2[i], 2);
        }
        return Math.sqrt(sum);
    }

    // 点积（向量归一化后等价于余弦相似度）
    public static double dotProduct(float[] v1, float[] v2) {
        double result = 0.0;
        for (int i = 0; i < v1.length; i++) {
            result += v1[i] * v2[i];
        }
        return result;
    }
}
```


#### 批量操作

```java
// 批量插入（提升性能）
public void batchInsert(List<Chunk> chunks) {
    jdbcTemplate.batchUpdate(
        "INSERT INTO document_chunks (document_id, chunk_text, embedding) VALUES (?, ?, ?)",
        new BatchPreparedStatementSetter() {
            @Override
            public void setValues(PreparedStatement ps, int i) throws SQLException {
                Chunk chunk = chunks.get(i);
                ps.setString(1, chunk.getDocumentId());
                ps.setString(2, chunk.getText());
                ps.setObject(3, new PGvector(chunk.getEmbedding()));
            }

            @Override
            public int getBatchSize() {
                return chunks.size();
            }
        }
    );
}
```

## 3. Agent（智能体）

### 🎯 掌握程度：必须掌握

### 什么是Agent？

**通俗解释：**
Agent就是"能主动做事的AI"。普通LLM只能对话，Agent能调用工具、执行任务、自主规划。

**技术定义：**
具备感知、推理、行动能力的AI系统，能够理解目标、规划步骤、调用工具、执行任务。

### Agent vs 传统LLM

```
传统LLM：
用户："帮我查一下北京今天天气"
LLM："我是AI，无法查询实时信息" ❌

Agent：
用户："帮我查一下北京今天天气"
Agent：
1. 理解目标：需要查天气
2. 规划步骤：调用天气API
3. 执行动作：调用 WeatherService.getWeather("北京")
4. 返回结果："北京今天晴天，15-25℃" ✅
```

```java
@Agent
public class MemoryAgent {

    private ConversationMemory shortTermMemory;

    // 长期记忆：向量数据库
    private VectorDatabase longTermMemory;

    @Memory
    public void remember(String sessionId, String information) {

        shortTermMemory.add(sessionId, information);

        // 重要信息存入长期记忆
        if (isImportant(information)) {
            float[] embedding = embeddingService.embed(information);
            longTermMemory.store(new Memory(
                sessionId,
                information,
                embedding
            ));
        }
    }

    @Memory
    public String recall(String sessionId, String query) {
        // 先查短期记忆
        String fromShortTerm = shortTermMemory.search(sessionId, query);
        if (fromShortTerm != null) {
            return fromShortTerm;
        }

        // 再查长期记忆
        float[] queryVector = embeddingService.embed(query);
        List<Memory> memories = longTermMemory.search(queryVector, 5);

        return memories.isEmpty() ? null : memories.get(0).getContent();
    }
```

### Agent架构设计

```java
@Service
public class AgentFramework {

    private LLMAIService llmService;
    private ToolRegistry toolRegistry;
    private MemoryManager memoryManager;

    // Agent执行主流程
    public AgentResponse execute(String userMessage) {

        // 1. 理解用户意图
        Intent intent = understandIntent(userMessage);

        // 2. 检索相关记忆
        List<Memory> relevantMemories = memoryManager.recall(
            userMessage,
            topK = 5
        );

        // 3. 规划执行步骤
        Plan plan = planExecution(intent, relevantMemories);

        // 4. 执行计划
        ExecutionResult result = executePlan(plan);

        // 5. 保存新记忆
        memoryManager.remember(userMessage, result);

        // 6. 生成回复
        return generateResponse(result);
    }

    // 规划执行
    private Plan planExecution(Intent intent, List<Memory> memories) {

        String prompt = String.format("""
            你是一个智能Agent。请为以下任务制定执行计划：

            【用户意图】
            %s

            【相关记忆】
            %s

            【可用工具】
            %s

            请生成执行计划，JSON格式：
            {
              "steps": [
                {"action": "tool_name", "parameters": {...}},
                {"action": "tool_name", "parameters": {...}}
              ],
              "reasoning": "规划理由"
            }
            """,
            intent,
            memories,
            toolRegistry.listTools()
        );

        String response = llmService.chat(prompt);
        return parsePlan(response);
    }

    // 执行计划
    private ExecutionResult executePlan(Plan plan) {

        List<StepResult> stepResults = new ArrayList<>();

        for (Step step : plan.getSteps()) {
            try {
                // 调用工具
                Tool tool = toolRegistry.getTool(step.getAction());
                Object result = tool.execute(step.getParameters());

                stepResults.add(new StepResult(
                    step,
                    Status.SUCCESS,
                    result
                ));

            } catch (Exception e) {
                // 执行失败，可能需要重新规划
                stepResults.add(new StepResult(
                    step,
                    Status.FAILED,
                    e.getMessage()
                ));

                // 简单的Agent：直接返回错误
                // 高级Agent：重新规划
                if (stepResults.size() > 3) {
                    break; // 防止无限重试
                }

                // 重新规划
                Plan newPlan = replan(plan, stepResults);
                return executePlan(newPlan);
            }
        }

        return new ExecutionResult(stepResults);
    }
}
```

## 4. Function Calling（函数调用）

### 🎯 掌握程度：必须掌握

### 什么是Function Calling？

**通俗解释：**
Function Calling让LLM能够"调用Java函数"。就像你告诉同事"帮我查数据库"，LLM可以"调用你的Java方法"。

**技术定义：**
LLM理解用户意图后，生成结构化的函数调用请求，由业务代码执行具体函数并返回结果。

### Function Calling vs Agent

```
Function Calling：
├─ 单次工具调用
├─ LLM生成函数名和参数
├─ 业务代码执行
└─ 返回结果给LLM

Agent：
├─ 多步骤任务
├─ 自主规划和决策
├─ 可能多次调用工具
└─ 更智能、更复杂
```

### Function Calling工作流程

```
┌─────────────────────────────────────────┐
│ Function Calling 完整流程                │
└─────────────────────────────────────────┘

1. 定义函数
   ┌──────────────────┐
   │ getWeather(city) │  Java函数
   └──────────────────┘

2. 告诉LLM有哪些函数
   ┌──────────────────────────┐
   │ 发送给LLM：              │
   │ {                        │
   │   "functions": [         │
   │     {                    │
   │       "name": "getWeather",│
   │       "parameters": {    │
   │         "city": "string"│
   │       }                  │
   │     }                    │
   │   ]                      │
   │ }                        │
   └──────────────────────────┘

3. 用户提问
   用户："北京今天天气怎么样？"

4. LLM决定调用函数
   ┌──────────────────────────┐
   │ LLM返回：                │
   │ {                        │
   │   "function_call": {     │
   │     "name": "getWeather",│
   │     "arguments": {       │
   │       "city": "北京"     │
   │     }                    │
   │   }                      │
   │ }                        │
   └──────────────────────────┘

5. 执行函数
   Java代码执行：getWeather("北京")
   返回："晴天，15-25℃"

6. 把结果发回LLM
   ┌──────────────────────────┐
   │ 发送给LLM：              │
   │ {                        │
   │   "function_result": {   │
   │     "name": "getWeather",│
   │     "result": "晴天，15-25℃"│
   │   }                      │
   │ }                        │
   └──────────────────────────┘

7. LLM生成最终回复
   "北京今天是晴天，气温15到25度，适合出行。"
```

```java
@Service
public class FunctionCallingService {

    private LLMAIService llmService;

    // 定义可调用的函数
    @FunctionDefinition
    public String getWeather(@Param("city") String city) {
        // 实际的天气查询逻辑
        return weatherService.getWeather(city);
    }

    @FunctionDefinition
    public List<Order> queryOrders(
        @Param("customerId") String customerId,
        @Param("status") String status
    ) {
        return orderRepository.findByCustomerIdAndStatus(
            customerId,
            status
        );
    }

    @FunctionDefinition
    public String sendEmail(
        @Param("to") String to,
        @Param("subject") String subject,
        @Param("body") String body
    ) {
        emailService.send(to, subject, body);
        return "邮件已发送";
    }

    // Function Calling主流程
    public String chat(String userMessage) {

        // 1. 获取所有函数定义
        List<FunctionDefinition> functions = getFunctionDefinitions();

        // 2. 调用LLM，决定是否调用函数
        LLResponse response = llmService.chat(
            userMessage,
            functions  // 告诉LLM有哪些函数可用
        );

        // 3. 如果LLM决定调用函数
        if (response.hasFunctionCall()) {

            FunctionCall functionCall = response.getFunctionCall();

            // 执行函数
            Object result = executeFunction(
                functionCall.getName(),
                functionCall.getArguments()
            );

            // 4. 把结果发回LLM，生成最终回复
            return llmService.generateResponse(
                userMessage,
                functionCall,
                result
            );
        }

        // 5. 如果不需要调用函数，直接返回LLM回复
        return response.getContent();
    }

    // 执行函数（反射）
    private Object executeFunction(String functionName,
                                   Map<String, Object> arguments) {
        try {
            // 找到对应的方法
            Method method = findMethod(functionName);

            // 准备参数
            Object[] args = prepareArguments(method, arguments);

            // 调用方法
            return method.invoke(this, args);

        } catch (Exception e) {
            return "执行失败：" + e.getMessage();
        }
    }
}
```

## 5. Chain of Thought（思维链）

### 🎯 掌握程度：必须掌握

### 什么是Chain of Thought？

**通俗解释：**
CoT就是让AI"一步步思考"，而不是直接给答案。就像做数学题要写过程，AI也需要"展示思考过程"才能得出正确答案。

**技术定义：**
通过提示词引导LLM生成中间推理步骤，从而提升复杂问题求解能力的提示技巧。

### CoT的效果对比

#### ❌ 直接回答（无CoT）

```
用户：如果小明有3个苹果，吃了1个，又买了5个，现在有几个？

LLM：小明现在有7个苹果。（可能正确，但没展示过程）
```

#### ✅ 思维链（CoT）

```
用户：如果小明有3个苹果，吃了1个，又买了5个，现在有几个？
请一步步思考。

LLM：
1. 小明最初有3个苹果
2. 吃了1个：3 - 1 = 2个
3. 又买了5个：2 + 5 = 7个
4. 所以小明现在有7个苹果。（展示推理过程，更可靠）
```

### CoT的三种模式

#### 模式1：Zero-Shot CoT

```java
String prompt = """
用户问题：%s

请一步步思考：
1. 首先...
2. 然后...
3. 最后...

最终答案：
""";
```

#### 模式2：Few-Shot CoT

```java

- 进货：7 + 5 = 12个

- 买书：100 - 30 - 30 = 40元

""";
```

#### 模式3：Self-Consistency

## 7. Knowledge Graph（知识图谱）

### 🚀 掌握程度：进阶学习

### 什么是知识图谱？

**通俗解释：**
知识图谱是"事物之间的关系网"。就像社交网络是"人与人的关系"，知识图谱是"概念与概念的关系"。

**技术定义：**
用图结构表示知识的系统，节点代表实体，边代表关系。

### 知识图谱示例

```
实体关系图：

[苹果公司] --(CEO)--> [蒂姆·库克]
   |
   |--(生产)--> [iPhone]
   |               |
   |               |--(发布于)--> [2007年]
   |               |--(属于)--> [智能手机]
   |
   |--(竞争对手)--> [三星电子]
```

### 知识图谱 vs RAG

```
RAG：
├─ 非结构化文本
├─ 向量相似度匹配
├─ 适合语义检索
└─ 优点：实现简单

知识图谱：
├─ 结构化知识
├─ 精确关系查询
├─ 适合复杂推理
└─ 优点：准确性高

最佳实践：RAG + 知识图谱结合
```

### 学习建议

```
3个月内：
├─ 了解知识图谱概念
├─ 不需要深入学习
└─ 专注于RAG技术

3个月后（进阶）：
├─ 根据业务需要
├─ 考虑引入知识图谱
└─ 与RAG结合使用
```
