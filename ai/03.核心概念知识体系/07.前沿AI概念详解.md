## 第一部分：Engineering方法论体系

### 1. Prompt Engineering（提示词工程） vs Context Engineering（上下文工程）

#### Prompt Engineering

**通俗解释：**
Prompt Engineering就是"学会和AI说话的艺术"。就像你指导新人一样，你说得越清楚，AI做得越好。

**核心要点：**
```
Prompt = 指令 + 上下文 + 约束 + 示例

好的Prompt特征：
✅ 明确的目标
✅ 充足的上下文
✅ 清晰的约束
✅ 具体的示例

优化技巧：
├─ 迭代优化：多轮改进
├─ A/B测试：对比效果
├─ 模板化：复用优秀Prompt
└─ 工具化：使用Prompt管理工具
```

**实际示例：**

```java
// ❌ 差的Prompt
String badPrompt = "写一个用户登录功能";

// ✅ 好的Prompt
String goodPrompt = """

    """;
```

#### Context Engineering

**通俗解释：**
Context Engineering就是"管理AI记忆的艺术"。就像你开会时需要提供背景资料，AI也需要合适的上下文才能做出好的决策。

**核心挑战：**
```
Context Window限制问题：
├─ 上下文有限（如4K、8K、32K Tokens）
├─ 超出限制会被截断
└─ 需要智能管理上下文信息

Context Engineering解决：
├─ 上下文选择：保留最相关的信息
├─ 上下文压缩：总结历史对话
├─ 上下文分层：短期+长期记忆
└─ 上下文优化：去除冗余信息
```

**技术实现：**

```java
@Service
public class ContextEngineeringService {

    // 1. 上下文选择策略
    public List<Message> selectRelevantContext(
        String sessionId,
        String currentQuery,
        int maxTokens
    ) {
        // 获取所有历史消息
        List<Message> allHistory = getHistory(sessionId);

        // 计算相关性
        List<Message> relevantMessages = allHistory.stream()
            .filter(msg -> isRelevant(msg, currentQuery))
            .collect(Collectors.toList());

        // 限制Token数量
        return limitTokens(relevantMessages, maxTokens);
    }

    // 2. 上下文压缩
    public String compressContext(List<Message> history) {

        // 使用LLM总结历史对话
        String summaryPrompt = String.format("""
            请总结以下对话的关键信息：

            %s

            总结要点：
            1. 用户的核心需求是什么？
            2. 讨论了哪些关键问题？
            3. 得出了什么结论？

            请以简洁的方式总结。
            """, history.toString()
        );

        return llmService.chat(summaryPrompt);
    }

    // 3. 分层上下文管理
    public Context buildHierarchicalContext(
        String sessionId,
        String query
    ) {
        Context context = new Context();

        // 短期上下文：最近5条消息
        context.addShortTerm(getRecentMessages(sessionId, 5));

        // 中期上下文：当前会话摘要
        context.addMediumTerm(getSessionSummary(sessionId));

        // 长期上下文：用户偏好、历史信息
        context.addLongTerm(getUserProfile(sessionId));

        // 动态上下文：根据查询检索相关信息
        context.addDynamic(getRelevantInfo(query));

        return context;
    }

    // 4. 上下文优化
    public List<Message> optimizeContext(List<Message> context) {

        // 去除冗余信息
        List<Message> deduplicated = removeDuplicates(context);

        // 合并相似内容
        List<Message> merged = mergeSimilar(deduplicated);

        // 按重要性排序
        List<Message> ranked = rankByImportance(merged);

        return ranked;
    }

    // Token预算管理
    public class TokenBudget {

        private final int maxTokens;
        private int usedTokens = 0;

        public TokenBudget(int maxTokens) {
            this.maxTokens = maxTokens;
        }

        public boolean canAdd(String text) {
            int tokens = estimateTokens(text);
            return (usedTokens + tokens) <= maxTokens;
        }

        public void add(String text) {
            if (canAdd(text)) {
                usedTokens += estimateTokens(text);
            }
        }

        public int getRemaining() {
            return maxTokens - usedTokens;
        }
    }
}
```

```java
// 使用TokenBudget管理上下文
public class SmartContextManager {

    public List<Message> buildOptimalContext(
        String sessionId,
        String query,
        int budget
    ) {
        TokenBudget tokenBudget = new TokenBudget(budget);
        List<Message> context = new ArrayList<>();

        // 1. 系统Prompt（必须，约200 tokens）
        String systemPrompt = getSystemPrompt();
        if (tokenBudget.canAdd(systemPrompt)) {
            context.add(new Message(MessageType.SYSTEM, systemPrompt));
            tokenBudget.add(systemPrompt);
        }

        // 2. 相关历史（按相关性排序，选择性添加）
        List<Message> history = getHistory(sessionId);
        List<ScoredMessage> scoredHistory = scoreRelevance(history, query);

        for (ScoredMessage item : scoredHistory) {
            if (tokenBudget.canAdd(item.message.getContent())) {
                context.add(item.message);
                tokenBudget.add(item.message.getContent());
            } else {
                break; // 预算用完
            }
        }

        // 3. 检索的上下文（RAG）
        if (tokenBudget.getRemaining() > 1000) {
            List<Document> docs = ragService.search(query, 5);
            String ragContext = formatRAGContext(docs);

            if (tokenBudget.canAdd(ragContext)) {
                context.add(new Message(MessageType.SYSTEM, ragContext));
                tokenBudget.add(ragContext);
            }
        }

        return context;
    }
}
```

### 2. Agentic Engineering（智能体工程）

#### 什么是Agentic Engineering？

**通俗解释：**
Agentic Engineering就是"设计和构建AI智能体的工程方法"。就像软件工程指导软件开发，Agentic Engineering指导Agent系统的构建。

**核心要素：**
```
Agentic Engineering = 需求分析 + 能力设计 + 工具集成 + 行为编排

与传统开发的区别：
传统开发：固定流程，明确分支
Agentic开发：自主决策，动态规划

关键能力设计：
├─ 感知能力（Perception）
│   └─ 理解用户意图
├─ 规划能力（Planning）
│   └─ 分解复杂任务
├─ 执行能力（Action）
│   └─ 调用工具完成任务
└─ 记忆能力（Memory）
    └─ 保存和检索信息
```

#### Agentic系统架构

```java
// Agent架构设计
public interface Agent {

    // 核心能力
    PerceptionResult perceive(String input);
    PlanResult plan(String goal);
    ActionResult execute(Action action);
    void remember(Memory memory);

    // 高级能力
    boolean shouldAskForHelp(String task);
    List<Tool> selectTools(String goal);
    String reflectOnResult(ActionResult result);
}

// Agentic系统实现
@Service
public class AgenticSystem implements Agent {

    @Autowired
    private ToolRegistry toolRegistry;

    @Autowired
    private MemoryManager memoryManager;

    @Autowired
    private LLMService llmService;

    @Override
    public PerceptionResult perceive(String input) {

        // 使用LLM理解用户意图
        String prompt = String.format("""
            分析以下用户输入的意图：

            输入：%s

            请识别：
            1. 用户想要什么？（goal）
            2. 需要什么信息？（context）
            3. 有什么约束？（constraints）

            返回JSON格式。
            """, input
        );

        String response = llmService.chat(prompt);
        return parsePerception(response);
    }

    @Override
    public PlanResult plan(String goal) {

        // 生成执行计划
        String prompt = String.format("""
            为以下目标制定执行计划：

            目标：%s

            可用工具：
            %s

            请生成：
            1. 执行步骤（step by step）
            2. 每步需要的工具
            3. 预期结果

            返回JSON格式的计划。
            """, goal, listAvailableTools()
        );

        String response = llmService.chat(prompt);
        return parsePlan(response);
    }

    @Override
    public ActionResult execute(Action action) {

        // 执行具体的Action
        Tool tool = toolRegistry.getTool(action.getToolName());

        try {
            Object result = tool.execute(action.getParameters());
            return ActionResult.success(result);

        } catch (Exception e) {
            // 执行失败，需要重新规划
            return ActionResult.failure(e.getMessage());
        }
    }

    @Override
    public void remember(Memory memory) {

        // 保存到短期记忆
        memoryManager.saveToShortTerm(memory);

        // 如果是重要信息，保存到长期记忆
        if (memory.isImportant()) {
            memoryManager.saveToLongTerm(memory);
        }
    }

    // Agent主循环
    public String run(String userRequest) {

        // 1. 感知
        PerceptionResult perception = perceive(userRequest);

        // 2. 规划
        PlanResult plan = plan(perception.getGoal());

        // 3. 执行（循环执行每个步骤）
        List<ActionResult> results = new ArrayList<>();

        for (Step step : plan.getSteps()) {

            log.info("执行步骤：{}", step.getDescription());

            ActionResult result = execute(step.getAction());

            if (result.isSuccess()) {
                results.add(result);
            } else {
                // 执行失败，重新规划
                log.warn("步骤执行失败：{}", result.getError());
                plan = replan(perception.getGoal(), results);
                continue;
            }
        }

        // 4. 生成最终回复
        return generateResponse(userRequest, results);
    }
}
```

#### Agentic Design Patterns

```java
// Pattern 1: ReAct模式（Reasoning + Acting）
@Service
public class ReActAgent {

    public String process(String query) {

        StringBuilder thoughtProcess = new StringBuilder();
        String current = query;

        for (int i = 0; i < 5; i++) { // 最多5轮思考

            // Thought: 思考当前状态
            String thought = llmService.think(current);
            thoughtProcess.append("Thought ").append(i + 1).append(": ").append(thought).append("\n");

            // Action: 决定下一步行动
            String action = llmService.decideAction(thought);

            if (action.equals("finish")) {
                // 完成任务
                return llmService.generateAnswer(thoughtProcess.toString());
            }

            // Observation: 执行行动并观察结果
            String observation = executeAction(action);
            thoughtProcess.append("Observation: ").append(observation).append("\n");

            // 更新当前状态
            current = observation;
        }

        return llmService.generateAnswer(thoughtProcess.toString());
    }
}

// Pattern 2: Reflexion模式（带反思的Agent）
@Service
public class ReflexionAgent {

    public String executeWithReflexion(String goal) {

        int maxAttempts = 3;

        for (int attempt = 1; attempt <= maxAttempts; attempt++) {

            // 执行任务
            ActionResult result = executeGoal(goal);

            if (result.isSuccess()) {
                return result.getOutput();
            }

            // 反思：分析失败原因
            String reflection = reflect(goal, result);

            // 修正：根据反思调整策略
            adjustStrategy(reflection);
        }

        return "抱歉，无法完成任务";
    }

    private String reflect(String goal, ActionResult result) {

        String prompt = String.format("""
            任务：%s
            结果：%s
            错误：%s

            请分析：
            1. 为什么失败？
            2. 应该如何改进？
            3. 下次应该注意什么？

            请提供反思和改进建议。
            """, goal, result.getOutput(), result.getError()
        );

        return llmService.chat(prompt);
    }
}
```

### 3. Harness Engineering（驾驭工程）

#### 什么是Harness Engineering？

**通俗解释：**
Harness Engineering就是"驾驭AI系统的工程艺术"。就像驯马师驯服野马，Harness Engineering让强大的AI模型可控、可靠、安全地工作。

**核心理念：**
```
Harness = Control + Reliability + Safety

Control（控制）：
├─ 输入控制：验证和过滤
├─ 输出控制：审核和限制
└─ 行为控制：约束和引导

Reliability（可靠性）：
├─ 重试机制：处理失败
├─ 降级策略：备用方案
└─ 监控告警：及时发现问题

Safety（安全性）：
├─ 安全边界：限制操作范围
├─ 人工审核：高风险操作
└─ 紧急停止：快速中断
```

#### Harness架构设计

```java
@Service
public class AIHarness {

    @Autowired
    private LLMAIService llmService;

    @Autowired
    private InputValidator inputValidator;

    @Autowired
    private OutputFilter outputFilter;

    @Autowired
    private SafetyMonitor safetyMonitor;

    // 被控制的AI调用
    public HarnessResult callLLM(String userInput) {

        // Stage 1: 输入控制
        ValidationResult validation = inputValidator.validate(userInput);
        if (!validation.isValid()) {
            return HarnessResult.rejected(validation.getReason());
        }

        // Stage 2: 安全检查
        SafetyCheck safetyCheck = safetyMonitor.check(userInput);
        if (safetyCheck.isUnsafe()) {
            return HarnessResult.blocked(safetyCheck.getReason());
        }

        // Stage 3: 执行（带重试）
        String rawOutput = executeWithRetry(userInput, 3);

        // Stage 4: 输出过滤
        String filteredOutput = outputFilter.filter(rawOutput);

        // Stage 5: 最终审核
        if (outputFilter.needsHumanReview(filteredOutput)) {
            return HarnessResult.needsReview(filteredOutput);
        }

        return HarnessResult.success(filteredOutput);
    }

    private String executeWithRetry(String input, int maxRetries) {

        for (int i = 0; i < maxRetries; i++) {

            try {
                return llmService.chat(input);

            } catch (RateLimitException e) {
                // 限流：等待后重试
                sleep(extractWaitTime(e));

            } catch (TimeoutException e) {
                // 超时：重试
                if (i == maxRetries - 1) {
                    throw e;
                }

            } catch (LLMException e) {
                // 其他错误：可能需要降级
                return fallbackStrategy(input);
            }
        }

        throw new RuntimeException("Max retries exceeded");
    }

    // 降级策略
    private String fallbackStrategy(String input) {

        log.warn("Primary LLM failed, using fallback");

        // 降级到备用模型
        if (fallbackLLM != null) {
            try {
                return fallbackLLM.chat(input);
            } catch (Exception e) {
                log.error("Fallback LLM also failed", e);
            }
        }

        // 最终降级：返回安全响应
        return "抱歉，服务暂时不可用，请稍后重试。";
    }
}
```

## 第二部分：Agent协作模式

### 1. A2A（Agent to Agent）

#### 什么是A2A？

**通俗解释：**
A2A就是"Agent之间如何协作"。就像团队中不同成员分工合作，多个Agent也需要相互配合完成复杂任务。

**协作模式：**
```
1. 层级协作（Hierarchical）
   Manager Agent
   ├── Worker Agent 1
   ├── Worker Agent 2
   └── Worker Agent 3

2. 平等协作（Flat）
   Agent A ←→ Agent B
      ↕          ↕
   Agent C ←→ Agent D

3. 专业化协作（Specialized）
   Research Agent → 搜索信息
   Writing Agent → 撰写内容
   Review Agent → 审核质量

4. 争论式协作（Debating）
   Agent A 提出方案
   Agent B 提出质疑
   Agent C 综合意见
```

#### A2A实现

```java
@Service
public class A2AOrchestrator {

    // 层级协作
    public String hierarchicalExecution(String goal) {

        // Manager Agent规划任务
        Agent manager = new ManagerAgent();
        Plan plan = manager.plan(goal);

        // 分配给Worker Agents
        List<WorkerAgent> workers = getWorkers();
        List<TaskResult> results = new ArrayList<>();

        for (Task task : plan.getTasks()) {

            // 选择合适的Worker
            WorkerAgent worker = selectBestWorker(workers, task);

            // 执行任务
            TaskResult result = worker.execute(task);
            results.add(result);

            // Manager根据结果调整后续任务
            if (!result.isSuccess()) {
                manager.rebalancePlan(plan, results);
            }
        }

        // Manager汇总结果
        return manager.synthesize(results);
    }

    // 平等协作
    public String flatCollaboration(String problem) {

        List<Agent> agents = List.of(
            new ResearchAgent(),
            new WritingAgent(),
            new CodingAgent()
        );

        // 并行生成解决方案
        List<Solution> solutions = agents.parallelStream()
            .map(agent -> agent.solve(problem))
            .collect(Collectors.toList());

        // 投票选择最佳方案
        Solution bestSolution = voteOnBest(solutions);

        return bestSolution.getContent();
    }

    // 专业化协作
    public String specializedCollaboration(String request) {

        // 1. Research Agent：收集信息
        ResearchAgent researcher = new ResearchAgent();
        ResearchResult info = researcher.gatherInfo(request);

        // 2. Writing Agent：生成内容
        WritingAgent writer = new WritingAgent();
        String draft = writer.generateContent(request, info);

        // 3. Review Agent：审核质量
        ReviewAgent reviewer = new ReviewAgent();
        ReviewResult review = reviewer.review(draft);

        if (review.needsRevision()) {
            // 迭代改进
            draft = writer.revise(draft, review.getFeedback());
        }

        return draft;
    }
}
```

### 2. A2UI（Agent to User Interface）

#### 什么是A2UI？

**通俗解释：**
A2UI就是"Agent如何与用户交互"。传统UI是图形界面，A2UI是对话式、智能的交互方式。

**交互模式：**
```
1. 对话式交互（Conversational）
   用户：用自然语言表达需求
   Agent：理解并执行任务

2. 主动式交互（Proactive）
   Agent：主动提供建议
   用户：确认或拒绝

3. 协作式交互（Collaborative）
   用户：提供方向
   Agent：给出选项
   用户：做出决策

4. 渐进式交互（Progressive）
   Agent：逐步展示工作过程
   用户：随时干预和调整
```

#### A2UI实现

```java
@Service
public class A2UIService {

    // 对话式交互
    public ChatResponse chat(String userMessage, String sessionId) {

        // 1. 理解用户意图
        Intent intent = intentAnalyzer.analyze(userMessage);

        // 2. 如果需要更多信息
        if (intent.isMissingInformation()) {

            // Agent主动提问
            String question = generateClarifyingQuestion(intent);

            return ChatResponse.question(
                "为了更好地帮助你，我需要了解：",
                question
            );
        }

        // 3. 执行任务
        ActionResult result = executeTask(intent);

        // 4. 返回结果（带进度展示）
        return ChatResponse.result(result);
    }

    // 主动式交互
    public void proactiveEngagement(String userId) {

        // Agent主动发现可帮助的场景
        UserProfile profile = getUserProfile(userId);

        // 基于用户上下文提供建议
        if (profile.isWorkingOnTask()) {

            Task currentTask = profile.getCurrentTask();

            // Agent分析可能的帮助
            List<Suggestion> suggestions = analyzeAndSuggest(currentTask);

            if (!suggestions.isEmpty()) {

                // 主动提供建议（但不打扰）
                sendProactiveSuggestion(userId, suggestions);
            }
        }
    }

    // 协作式交互
    public InteractiveResponse collaborate(String goal) {

        // 1. Agent分析任务
        TaskAnalysis analysis = agent.analyze(goal);

        // 2. 生成多个选项
        List<Option> options = agent.generateOptions(analysis);

        // 3. 展示给用户选择
        return InteractiveResponse.options(
            "我分析了你的需求，有几个方案：",
            options
        );
    }

    // 渐进式交互
    public Flux<ProgressUpdate> executeWithProgress(String goal) {

        return Flux.create(sink -> {

            // Agent制定计划
            Plan plan = agent.plan(goal);

            // 逐步执行，实时汇报
            for (Step step : plan.getSteps()) {

                sink.next(ProgressUpdate.start(step));

                ActionResult result = agent.execute(step);

                if (result.isSuccess()) {
                    sink.next(ProgressUpdate.complete(step, result));
                } else {
                    sink.next(ProgressUpdate.fail(step, result.getError()));
                }
            }

            sink.complete();
        });
    }
}
```

```java
// 前端集成示例（JavaScript）
class AgentUI {

    async chat(agent, userMessage) {

        // 1. 发送消息
        const response = await agent.chat(userMessage);

        // 2. 处理不同类型的响应
        switch (response.type) {

            case 'question':
                // Agent需要更多信息
                this.showClarificationQuestion(response.question);
                break;

            case 'options':
                // Agent提供多个选项
                this.showOptions(response.options);
                break;

            case 'progress':
                // Agent逐步执行任务
                this.streamProgress(response.progress);
                break;

            case 'result':
                // Agent返回最终结果
                this.showResult(response.result);
                break;
        }
    }

    // 渐进式展示Agent工作过程
    streamProgress(progressStream) {

        progressStream.onUpdate((update) => {

            this.updateProgress(update);

            // 允许用户随时中断
            if (this.userCancelled) {
                progressStream.cancel();
            }
        });
    }
}
```

## 第三部分：通信协议标准

### 1. MCP（Model Context Protocol）

#### 什么是MCP？

**通俗解释：**
MCP就是"AI应用之间通信的标准语言"。就像API让Web服务通信，MCP让AI应用能够互相连接和协作。

**技术定义：**
Model Context Protocol是一个开放标准，定义了AI应用之间如何共享上下文、交换信息和调用工具。

**核心价值：**
```
标准化之前：
├─ 每个AI应用各自为政
├─ 集成困难，成本高
└─ 无法互通互操作

标准化之后（MCP）：
├─ 统一的通信协议
├─ 即插即用
└─ 生态系统繁荣

就像USB统一了外设接口，
MCP统一了AI应用的连接方式
```

#### MCP架构

```
┌─────────────────────────────────────┐
│         MCP Client                  │
│     (如：IDE、编辑器、应用)          │
└──────────┬──────────────────────────┘
           │ MCP Protocol
           ├─ Context Sharing
           ├─ Tool Invocation
           └─ Resource Access
┌──────────┴──────────────────────────┐
│         MCP Server                  │
│     (如：AI服务、Agent系统)         │
└─────────────────────────────────────┘
```

#### MCP实现示例

```java
// MCP Client实现
@Service
public class MCPClient {

    private final MCPTransport transport;

    // 连接到MCP Server
    public void connect(String serverUrl) {

        // 握手
        HandshakeRequest handshake = HandshakeRequest.builder()
            .protocolVersion("1.0")
            .clientName("MyIDE")
            .capabilities(List.of(
                "context",
                "tools",
                "resources"
            ))
            .build();

        HandshakeResponse response = transport.handshake(serverUrl, handshake);

        if (!response.isSuccess()) {
            throw new MCPConnectionException("握手失败");
        }
    }

    // 共享上下文到MCP Server
    public void shareContext(String contextId, Map<String, Object> context) {

        ContextMessage message = ContextMessage.builder()
            .contextId(contextId)
            .metadata(context)
            .timestamp(Instant.now())
            .build();

        transport.send(message);
    }

    // 调用MCP Server的工具
    public ToolResult invokeTool(String toolName, Map<String, Object> params) {

        ToolInvocationMessage message = ToolInvocationMessage.builder()
            .toolName(toolName)
            .parameters(params)
            .requestId(UUID.randomUUID().toString())
            .build();

        transport.send(message);

        // 等待响应
        return transport.waitForResponse(message.getRequestId());
    }

    // 访问MCP Server的资源
    public Resource accessResource(String resourceUri) {

        ResourceRequestMessage message = ResourceRequestMessage.builder()
            .uri(resourceUri)
            .method("GET")
            .build();

        transport.send(message);

        ResourceResponse response = transport.waitForResponse(
            message.getRequestId()
        );

        return response.getResource();
    }
}

// MCP Server实现
@Service
public class MCPServer {

    private final MCPTransport transport;

    // 处理MCP Client连接
    @EventListener
    public void onClientConnect(ConnectionEvent event) {

        log.info("MCP Client connected: {}", event.getClientId());

        // 发送欢迎消息和Server能力
        ServerAnnouncement announcement = ServerAnnouncement.builder()
            .serverName("AI Assistant")
            .version("1.0")
            .capabilities(List.of(
                "code-generation",
                "code-analysis",
                "refactoring"
            ))
            .availableTools(getAvailableTools())
            .build();

        transport.send(event.getClientId(), announcement);
    }

    // 处理上下文共享
    @EventListener
    public void onContextMessage(ContextMessage message) {

        // 存储上下文
        contextStore.store(message.getContextId(), message.getMetadata());

        log.debug("Context stored: {}", message.getContextId());
    }

    // 处理工具调用
    @EventListener
    public void onToolInvocation(ToolInvocationMessage message) {

        String toolName = message.getToolName();
        Map<String, Object> params = message.getParameters();

        try {
            // 查找并执行工具
            Tool tool = toolRegistry.getTool(toolName);
            Object result = tool.execute(params);

            // 返回结果
            ToolResponse response = ToolResponse.builder()
                .requestId(message.getRequestId())
                .success(true)
                .result(result)
                .build();

            transport.send(message.getClientId(), response);

        } catch (Exception e) {

            // 返回错误
            ToolResponse errorResponse = ToolResponse.builder()
                .requestId(message.getRequestId())
                .success(false)
                .error(e.getMessage())
                .build();

            transport.send(message.getClientId(), errorResponse);
        }
    }

    // 处理资源访问
    @EventListener
    public void onResourceRequest(ResourceRequestMessage message) {

        String uri = message.getUri();

        try {
            Resource resource = resourceStore.get(uri);

            ResourceResponse response = ResourceResponse.builder()
                .requestId(message.getRequestId())
                .success(true)
                .resource(resource)
                .build();

            transport.send(message.getClientId(), response);

        } catch (ResourceNotFoundException e) {

            ResourceResponse errorResponse = ResourceResponse.builder()
                .requestId(message.getRequestId())
                .success(false)
                .error("Resource not found: " + uri)
                .build();

            transport.send(message.getClientId(), errorResponse);
        }
    }
}
```

### 2. Skills（技能）系统

#### 什么是Skills？

**通俗解释：**
Skills就是"Agent的能力模块"。就像人类有不同的技能（编程、写作、设计），Agent也可以有各种可组合的技能。

**技术定义：**
Skill是Agent能力的封装单元，可以被多个Agent复用和组合。

**技能类型：**
```
1. 基础技能（Basic Skills）
   ├── 读写文件
   ├── 调用API
   └── 执行命令

2. 专业技能（Specialized Skills）
   ├── 代码生成
   ├── 文档编写
   ├── 数据分析
   └── 问题诊断

3. 复合技能（Composite Skills）
   ├── RAG技能（检索+生成）
   ├── 研发技能（代码+文档）
   └── 分析技能（数据+报告）
```

#### Skills实现

```java
// Skill接口
public interface Skill {

    String getName();
    String getDescription();
    boolean canExecute(String task);
    SkillResult execute(String task, Map<String, Object> params);
}

// 具体Skill实现
@Component
public class CodeGenerationSkill implements Skill {

    @Autowired
    private LLMAIService llmService;

    @Override
    public String getName() {
        return "code-generation";
    }

    @Override
    public String getDescription() {
        return "根据需求生成代码";
    }

    @Override
    public boolean canExecute(String task) {

        // 判断任务是否需要代码生成
        return task.toLowerCase().contains("生成") ||
               task.toLowerCase().contains("写") ||
               task.toLowerCase().contains("实现");
    }

    @Override
    public SkillResult execute(String task, Map<String, Object> params) {

        String prompt = buildPrompt(task, params);
        String code = llmService.chat(prompt);

        // 代码后处理
        code = postProcess(code);

        return SkillResult.success(code);
    }
}

// Skill Registry
@Service
public class SkillRegistry {

    private final Map<String, Skill> skills = new ConcurrentHashMap<>();

    @PostConstruct
    public void init() {

        // 注册所有Skill
        register(new CodeGenerationSkill());
        register(new DocumentationSkill());
        register(new DataAnalysisSkill());
        register(new DebuggingSkill());
    }

    public void register(Skill skill) {
        skills.put(skill.getName(), skill);
    }

    public Skill findBestSkill(String task) {

        return skills.values().stream()
            .filter(skill -> skill.canExecute(task))
            .findFirst()
            .orElse(null);
    }

    public List<Skill> findRelevantSkills(String task) {

        return skills.values().stream()
            .filter(skill -> skill.canExecute(task))
            .collect(Collectors.toList());
    }
}

// Agent使用Skills
@Service
public class SkillfulAgent {

    @Autowired
    private SkillRegistry skillRegistry;

    public String executeTask(String task) {

        // 1. 查找合适的Skill
        Skill skill = skillRegistry.findBestSkill(task);

        if (skill == null) {
            return "抱歉，我没有相关的技能来完成这个任务";
        }

        // 2. 提取参数
        Map<String, Object> params = extractParameters(task);

        // 3. 执行Skill
        SkillResult result = skill.execute(task, params);

        // 4. 返回结果
        return result.getOutput();
    }

    // 复合Skill：组合多个Skill
    public String executeCompositeTask(String task) {

        List<Skill> skills = skillRegistry.findRelevantSkills(task);

        // 按顺序执行多个Skill
        List<SkillResult> results = new ArrayList<>();

        for (Skill skill : skills) {

            SkillResult result = skill.execute(task, Map.of());
            results.add(result);

            // 根据前一个Skill的结果调整参数
            task = refineTask(task, result);
        }

        // 综合结果
        return synthesizeResults(results);
    }
}
```
